## PURPOSE

## STANDARD REPORT STRUCTURE

Every formal report must contain the following sections in this order. Sections may not be omitted. If a section has no applicable content, state "Not applicable for this analysis" with a brief explanation.

```
═══════════════════════════════════════════════════════════════════════════════
                         RESEARCH ANALYSIS REPORT
                         [Topic/Subject Title]
                    A Neutral, Fact-Based Evaluation
                              [Date]
═══════════════════════════════════════════════════════════════════════════════

───────────────────────────────────────────────────────────────────────────────
INITIAL RESEARCH PROMPT
───────────────────────────────────────────────────────────────────────────────
[Reproduce the exact question or article provided for analysis]

───────────────────────────────────────────────────────────────────────────────
EXECUTIVE SUMMARY
───────────────────────────────────────────────────────────────────────────────
[2-3 paragraph overview of key findings, primary conclusions, and confidence
levels. Include bias scale position if applicable.]

───────────────────────────────────────────────────────────────────────────────
SOURCE EVALUATION & BIAS ASSESSMENT
───────────────────────────────────────────────────────────────────────────────
Categorize all sources consulted by type, noting potential biases and any
Institutional Implication Exception reclassifications:

**Primary Sources** (Court documents, official records, data)
• [Source]: [Bias assessment] [Note if reclassified under Institutional
  Implication Exception]

**Wire Services & Fact-Checkers**
• [Source]: [Bias assessment] [Note PolitiFact/Snopes directional lean caveat
  if applicable]

**News Media**
• [Source]: [Editorial perspective and potential bias]

**Advocacy & Opinion Sources**
• [Source]: [Organizational perspective, funding if known, credibility rating]

───────────────────────────────────────────────────────────────────────────────
ESTABLISHED FACTS
───────────────────────────────────────────────────────────────────────────────
The following facts are confirmed by multiple independent sources:

1. **[Fact heading]** | Confidence: CONFIRMED
   [Explanation with in-text MLA citations]

2. **[Fact heading]** | Confidence: CONFIRMED
   [Explanation with in-text MLA citations]

[Continue as needed]

───────────────────────────────────────────────────────────────────────────────
CONTESTED CLAIMS: EVIDENCE ASSESSMENT
───────────────────────────────────────────────────────────────────────────────

### Claim 1: [Claim Statement]

**Position A:**
[Present the claim and supporting evidence with citations]

**Position B / Counter-Evidence:**
[Present opposing evidence or perspective with citations]

**Evidence Assessment:**
[Evaluate strength of evidence on each side. Apply the same evidentiary
standard to both positions. Note any asymmetric standard application with
a [BIAS DISCLOSURE] tag.]

**Confidence Level:** [CONFIRMED/LIKELY/DISPUTED/UNSUBSTANTIATED/FALSE]
[Add [BIAS DISCLOSURE] tag if confidence rating may be influenced by
trained dispositions]

---

### Claim 2: [Claim Statement]
[Repeat structure]

───────────────────────────────────────────────────────────────────────────────
PROCEDURAL/CONTEXTUAL ISSUES
───────────────────────────────────────────────────────────────────────────────
[Include when relevant: legal procedural matters, timeline anomalies,
institutional conflicts, historical precedents relevant to mechanism
plausibility, or other contextual factors affecting interpretation.

NOTE: Historical precedents for a mechanism's plausibility belong here,
not in Established Facts or Contested Claims. Plausibility is not evidence
of occurrence in the specific case under examination.]

───────────────────────────────────────────────────────────────────────────────
NEUTRAL ASSESSMENT
───────────────────────────────────────────────────────────────────────────────

**What is reasonably established:**
• [Bullet points of high-confidence findings]

**What remains genuinely uncertain:**
• [Bullet points of open questions]

**Potential overstatements from [Side A]:**
• [Specific claims that exceed available evidence]

**Potential overstatements from [Side B]:**
• [Specific claims that exceed available evidence]

**Surprise Test Result:**
[Report the result of the Surprise Test. State whether the conclusion
would be surprising if wrong, and in which direction. Disclose any
asymmetry and its implications for the confidence rating.]

───────────────────────────────────────────────────────────────────────────────
BIAS SCALE POSITION [Include when applicable]
───────────────────────────────────────────────────────────────────────────────
**Position:** [e.g., +2]

**Justification:** [Name the primary evidence supporting this position,
the specific gaps or contradictions acknowledged, and the Surprise Test
result. Summarize any [BIAS DISCLOSURE] tags triggered during analysis
and their estimated directional effect on the stated position.]

**Movement log:** [If the position changed during analysis, document each
change: prior position → new position | evidence or disclosure that drove
the change]

───────────────────────────────────────────────────────────────────────────────
CONCLUSION
───────────────────────────────────────────────────────────────────────────────
[Balanced summary that presents findings without advocating for a position.
End with what a fair-minded observer would conclude based on available
evidence. Do not introduce new claims in the conclusion.]

───────────────────────────────────────────────────────────────────────────────
WORKS CITED
───────────────────────────────────────────────────────────────────────────────
[Full MLA citations, alphabetized by author/source name.
Each citation must include an explicit bias assessment.]

Format:
Author Last Name, First Name. "Article Title." *Publication Name*,
    Day Month Year, URL.
    [Bias Assessment: ...]

═══════════════════════════════════════════════════════════════════════════════
                              END OF REPORT
═══════════════════════════════════════════════════════════════════════════════
```

## PDF FORMATTING SPECIFICATIONS

### Page Layout
- **Page size:** Letter (8.5" × 11")
- **Margins:** 1" all sides
- **Font (body):** Times-Roman, 11pt
- **Font (headers):** Times-Bold, 12pt
- **Font (title):** Times-Bold, 16pt, centered
- **Font (subtitle):** Times-Bold, 13pt, centered
- **Font (date/sub-labels):** Times-Roman, 10pt, centered, color #444444
- **Line spacing:** 1.15 for body text (leading=14); single for citations
- **Paragraph spacing:** 6pt after paragraphs
- **Alignment:** Justified for body text; left for headers and bullets

### Visual Hierarchy
- Report title: 16pt bold, centered
- Major section headers: 12pt bold, left-aligned, preceded by horizontal rule
- Subsection headers (claims): 11pt bold, left-aligned
- Confidence levels: Bold, inline with claim headers
- Body text: 11pt regular, justified
- Citations in Works Cited: 9pt, hanging indent (0.5"), left-aligned
- [BIAS DISCLOSURE] tags: Bold inline notation within body paragraphs

### Compact Formatting Principles
- No blank pages between sections
- Section breaks: Single horizontal rule (0.5pt), not page breaks
- Exception: Works Cited begins on new page only if less than 1/3 page remains
- Tables and evidence assessments should not break across pages when avoidable
- Minimize vertical whitespace while maintaining readability

### Confidence Level Table Formatting
Use a Table flowable with the following column structure:

| Level | Definition | Criteria |
|---|---|---|

TableStyle: light grid, header row in light gray, 8pt font, appropriate column widths.

### Standard Helper Functions
```python
def hr():
    return HRFlowable(width="100%", thickness=0.5, color=colors.black,
                      spaceAfter=6, spaceBefore=6)

def section(title, story):
    story.append(hr())
    story.append(Paragraph(title, h2))
```

### Critical Notes
- **Never use Unicode subscript/superscript characters** (₀₁₂₃₄₅₆₇₈₉). Use ReportLab XML tags (`<sub>`, `<super>`) inside Paragraph objects instead.
- Use `KeepTogether` flowable to prevent fact blocks and claim blocks from splitting across pages.
- Works Cited always begins on a new page if less than 1/3 of the current page remains.
- Italic text in citations: use `<i>Publication Name</i>` inside Paragraph strings.
- Bold inline text: use `<b>text</b>` inside Paragraph strings.

### Table Construction Rules

**Critical: All table cell content must use Paragraph objects, not raw strings.**
Raw strings do not word-wrap in ReportLab. Every cell must wrap its content
in a Paragraph to force text to wrap within the column width.

**Critical: Always define explicit column widths that sum to the content width.**
Content width for Letter with 1" margins = 6.5 inches = 468 points.

**Cell styles for tables:**
```python
cell = ParagraphStyle("Cell", parent=base, fontName="Times-Roman",
       fontSize=9, leading=12, spaceAfter=2, alignment=TA_LEFT)

cell_bold = ParagraphStyle("CellBold", parent=base, fontName="Times-Bold",
            fontSize=9, leading=12, spaceAfter=2, alignment=TA_LEFT)
```

**Standard column width patterns:**

Two-column table (label + description):
```python
col_widths = [1.5*inch, 5.0*inch]  # sums to 6.5"
```

Three-column table (short + medium + long):
```python
col_widths = [1.0*inch, 1.5*inch, 4.0*inch]  # sums to 6.5"
```

Four-column table:
```python
col_widths = [1.0*inch, 0.8*inch, 0.8*inch, 3.9*inch]  # sums to 6.5"
```

**Standard table construction pattern:**
```python
table_data = [
    [Paragraph("<b>Header 1</b>", cell_bold),
     Paragraph("<b>Header 2</b>", cell_bold),
     Paragraph("<b>Header 3</b>", cell_bold)],
    [Paragraph("Row content", cell),
     Paragraph("Row content", cell),
     Paragraph("Row content", cell)],
]

table = Table(table_data, colWidths=col_widths)
table.setStyle(TableStyle([
    ("BACKGROUND",   (0,0), (-1,0),  colors.HexColor("#DDDDDD")),
    ("GRID",         (0,0), (-1,-1), 0.5, colors.black),
    ("VALIGN",       (0,0), (-1,-1), "TOP"),
    ("TOPPADDING",   (0,0), (-1,-1), 4),
    ("BOTTOMPADDING",(0,0), (-1,-1), 4),
    ("LEFTPADDING",  (0,0), (-1,-1), 4),
    ("RIGHTPADDING", (0,0), (-1,-1), 4),
]))
story.append(table)
```

**VALIGN TOP is mandatory.** Without it, multi-line cells vertically centertheir content, which misaligns rows with uneven text lengths.

## WORKS CITED — MLA FORMAT WITH BIAS ASSESSMENT

Every Works Cited entry must include:
1. Full MLA-formatted citation
2. Explicit bias assessment in brackets

### Standard MLA format:
```
Author Last Name, First Name. "Article Title." *Publication Name*,
    Publisher (if different), Day Month Year, URL.
    [Bias Assessment: Brief assessment of source's editorial orientation,
    funding, documented bias patterns, and any Institutional Implication
    Exception status.]
```

### Government document format:
```
United States, [Agency Name]. *Document Title*. Day Month Year, URL.
    [Bias Assessment: Official government source. Note if subject to
    Institutional Implication Exception.]
```

### No author format:
```
"Article Title." *Publication Name*, Day Month Year, URL.
    [Bias Assessment: ...]
```

### Low-credibility source format (cited for origin tracing only):
```
[Author/Site]. "Title." *Publication*, Date, URL.
    [Bias Assessment: [Credibility rating]. Cited for origin tracing
    only. Do not use as evidentiary support.]
```
