## STANDARD REPORT STRUCTURE

Every formal report must contain the following sections in this order. Sections may not be omitted. If a section has no applicable content, state "Not applicable for this analysis" with a brief explanation.


## PDF FORMATTING SPECIFICATIONS

### Page Layout
- **Page size:** Letter (8.5" × 11")
- **Margins:** 1" all sides
- **Font (body):** Times-Roman, 11pt
- **Font (headers):** Times-Bold, 12pt
- **Font (title):** Times-Bold, 16pt, centered
- **Font (subtitle):** Times-Bold, 13pt, centered
- **Font (date/sub-labels):** Times-Roman, 10pt, centered, color #444444
- **Line spacing:** 1.15 for body text (leading=14); single for citations
- **Paragraph spacing:** 6pt after paragraphs
- **Alignment:** Justified for body text; left for headers and bullets

### Visual Hierarchy
- Report title: 16pt bold, centered
- Major section headers: 12pt bold, left-aligned, preceded by horizontal rule
- Subsection headers (claims): 11pt bold, left-aligned
- Confidence levels: Bold, inline with claim headers
- Body text: 11pt regular, justified
- Citations in Works Cited: 9pt, hanging indent (0.5"), left-aligned
- [BIAS DISCLOSURE] tags: Bold inline notation within body paragraphs

### Compact Formatting Principles
- No blank pages between sections
- Section breaks: Single horizontal rule (0.5pt), not page breaks
- Exception: Works Cited begins on new page only if less than 1/3 page remains
- Tables and evidence assessments should not break across pages when avoidable
- Minimize vertical whitespace while maintaining readability

### Confidence Level Table Formatting
Use a Table flowable with the following column structure:

| Level | Definition | Criteria |
|---|---|---|

TableStyle: light grid, header row in light gray, 8pt font, appropriate column widths.

### Critical Notes
- **Never use Unicode subscript/superscript characters** (₀₁₂₃₄₅₆₇₈₉). Use ReportLab XML tags (`<sub>`, `<super>`) inside Paragraph objects instead.
- Use `KeepTogether` flowable to prevent fact blocks and claim blocks from splitting across pages.
- Works Cited always begins on a new page if less than 1/3 of the current page remains.
- Italic text in citations: use `<i>Publication Name</i>` inside Paragraph strings.
- Bold inline text: use `<b>text</b>` inside Paragraph strings.

### Table Construction — Complete Implementation

Add these two style definitions immediately after the existing style
definitions block. They are mandatory for all tables:

```python
cell = ParagraphStyle("Cell", parent=base, fontName="Times-Roman",
       fontSize=9, leading=12, spaceAfter=2, alignment=TA_LEFT)

cell_bold = ParagraphStyle("CellBold", parent=base, fontName="Times-Bold",
            fontSize=9, leading=12, spaceAfter=2, alignment=TA_LEFT)
```

Add this complete table builder function immediately after the hr() and
section() helper functions. Use it for EVERY table in the report without
exception:

```python
def make_table(headers, rows, col_widths):
    """
    Build a ReportLab table with guaranteed word wrap and no cell overflow.
    
    headers: list of header strings e.g. ["Source", "Tier", "Notes"]
    rows: list of lists of strings e.g. [["FBI Report", "3", "Implicated"]]
    col_widths: list of inch values that MUST sum to 6.5
                e.g. [2.0*inch, 0.75*inch, 3.75*inch]
    
    COLUMN WIDTH RULE: values must sum to exactly 6.5*inch (the content
    width for Letter paper with 1" margins). If they do not sum to 6.5,
    content will overflow or leave dead space.
    """
    # Build header row using Paragraph objects — never raw strings
    header_row = [Paragraph(h, cell_bold) for h in headers]
    
    # Build data rows using Paragraph objects — never raw strings
    data_rows = []
    for row in rows:
        data_rows.append([Paragraph(str(cell_val), cell) 
                          for cell_val in row])
    
    table_data = [header_row] + data_rows
    
    t = Table(table_data, colWidths=col_widths, repeatRows=1)
    t.setStyle(TableStyle([
        ("BACKGROUND",    (0, 0), (-1, 0),  colors.HexColor("#DDDDDD")),
        ("GRID",          (0, 0), (-1, -1), 0.5, colors.black),
        ("VALIGN",        (0, 0), (-1, -1), "TOP"),
        ("TOPPADDING",    (0, 0), (-1, -1), 4),
        ("BOTTOMPADDING", (0, 0), (-1, -1), 4),
        ("LEFTPADDING",   (0, 0), (-1, -1), 4),
        ("RIGHTPADDING",  (0, 0), (-1, -1), 4),
        ("FONTSIZE",      (0, 0), (-1, -1), 9),
        ("ROWBACKGROUNDS",(0, 1), (-1, -1), 
         [colors.white, colors.HexColor("#F5F5F5")]),
    ]))
    return t
```

### Standard Column Width Configurations

ALWAYS use one of these configurations. Choose based on number of columns
and content type. Values are pre-calculated to sum to 6.5 inches.

```python
# 2 columns — label + long description
COL_2_LABEL_DESC    = [1.5*inch, 5.0*inch]

# 3 columns — source + short field + long notes
COL_3_SOURCE_NOTES  = [2.5*inch, 0.75*inch, 3.25*inch]

# 3 columns — source + bias + content bias (political bias table)
COL_3_BIAS_TABLE    = [2.75*inch, 0.75*inch, 0.75*inch, 2.25*inch]

# 4 columns — source + tier + bias + notes
COL_4_SOURCE_EVAL   = [2.0*inch, 0.6*inch, 0.7*inch, 3.2*inch]

# 5 columns — scale table
COL_5_SCALE         = [0.6*inch, 1.4*inch, 4.5*inch]
```

### How to Call the Function

```python
# Source evaluation table example
story.append(make_table(
    headers=["Source", "Tier", "Bias Assessment"],
    rows=[
        ["Rogers Commission Report (1986)", 
         "Tier 3*", 
         "IIE applied. Presidential commission; NASA is an implicated party."],
        ["Boisjoly Memo, July 31 1985", 
         "Tier 1", 
         "Internal document. Anti-institutional. No political bias."],
    ],
    col_widths=COL_3_SOURCE_NOTES
))

# Political bias table example
story.append(make_table(
    headers=["Source", "Source Bias", "Content Bias", "Notes"],
    rows=[
        ["Rogers Commission", "0", "0", 
         "Technically neutral presentation. IIE applied."],
    ],
    col_widths=COL_3_BIAS_TABLE
))
```

### ABSOLUTE RULES — Never Violate These

1. NEVER pass raw strings as table cell values. Always use Paragraph().
   The make_table() function handles this automatically — use it.

2. NEVER omit col_widths. A table without explicit column widths WILL
   overflow regardless of content length.

3. NEVER use col_widths that do not sum to 6.5*inch. Measure twice.

4. ALWAYS use make_table() for every table in every report. Do not write
   custom table construction code. The function exists to prevent the
   exact overflow errors that occur when tables are built ad hoc.

5. If a cell value contains special characters (<, >, &), escape them:
   Replace & with &amp;  Replace < with &lt;  Replace > with &gt;

## WORKS CITED — MLA FORMAT WITH BIAS ASSESSMENT

Every Works Cited entry must include:
1. Full MLA-formatted citation
2. Explicit bias assessment in brackets

### Standard MLA format:
```
Author Last Name, First Name. "Article Title." *Publication Name*,
    Publisher (if different), Day Month Year, URL.
    [Bias Assessment: Brief assessment of source's editorial orientation,
    funding, documented bias patterns, and any Institutional Implication
    Exception status.]
```

### Government document format:
```
United States, [Agency Name]. *Document Title*. Day Month Year, URL.
    [Bias Assessment: Official government source. Note if subject to
    Institutional Implication Exception.]
```

### No author format:
```
"Article Title." *Publication Name*, Day Month Year, URL.
    [Bias Assessment: ...]
```

### Low-credibility source format (cited for origin tracing only):
```
[Author/Site]. "Title." *Publication*, Date, URL.
    [Bias Assessment: [Credibility rating]. Cited for origin tracing
    only. Do not use as evidentiary support.]
```
